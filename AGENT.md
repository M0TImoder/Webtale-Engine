# AGEND.md

あなたはWebTale-Engine(以降: プロジェクト)において開発を行うAIエージェントである。

## 規則

1. コメントアウトは日本語で必要最小限かつ、簡潔に行う必要がある

- コメントアウトを文章で書かない
- 句点を用いない

  ```悪い例
  // ここはダメージ倍率を指定する箇所です。
  // --- ここから変更開始 ---
  const float damageMagnification = 1.0; // ★★★ changed 2.0 to 1.0.
  /// --- ここまで変更 ---
  ```

  ```いい例
  // ダメージ倍率
  const float damageMagnification = 1.0;
  ```

2. 命名法は簡潔かつロワーキャメル法に統一する必要がある

  ```悪い例
  Damage_Magnification_Of_Enemy
  ```

  ```いい例
  enemyDamageMag
  ```

3. エラーが解決するまで継続してコードを改善すること

- 「迅速に仕事を終える事」を意識してはならない
- 「エラーが無い」「指示を的確に反映した」「可読性の高い」「最高品質のコード」を提供する必要がある

4. 独自の解釈でコードを書いてはならない

- ユーザーからの指示が曖昧な場合、必ず「つまり○○ということでしょうか？」などと質問を行い、意図を完璧に把握するまでコードを記述してはならない

5. ファイルを読み込む際はgrepを使用せず、存在するファイル(.gitやnode_modules, ビルド生成物等を除く)の中身を一括で全てロードし読み込むこと
